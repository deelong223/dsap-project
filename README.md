# [專題名稱]
抽卡概率率計算機
## Proposal Report

### 動機與目標
1.遊玩含抽卡的線上遊戲時，因規則複雜而容易忽視抽卡資源本身的價值。在累計抽數不繼承的抽卡遊戲中，常會有因一時衝動消費，又沒抽到
獎勵，進而使投入進去的資源全變為無效資源的情況。
2.玩家在玩抽卡遊戲時，常因局部的稀有情況而高估/低估整體的運勢。通過過往歷史分析，能夠得出較為客觀的結論。
### 預期功能
1.在固定遊戲(規則與概率分布確定)的情況下，計算在給定資訊的條件下，達成事件的概率。
2.輸入所有相關變因，如預算，要抽取哪些與其數量等等。在固定規則的情況下，計算出在該條件下，能成功獲得輸入的目標之機率。
3.舉例:某卡池規定如下
(a)if count=b，抽取事件發生時，有P(b)的機率(a++,b=0)，1-P(b)的機率(b++)
(b)初始時a=0，b=0
(c)如果要在抽取這個事件發生n次，a>=m的機率為何?

### 使用技術
C++

### 時程規劃

第13周前完成project

在我要做的遊戲中，可以將問題從指數級時間縮小到多項式時間。
